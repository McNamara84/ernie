# Findings from Repository Analysis

## Generic Accept Flow

Suggestions are accepted via:

```php
acceptSuggestion(int $id)
```

The `GenericTableAssistant` loads the suggestion by ID and delegates the actual update logic to:

```php
applyAccepted(AssistantSuggestion $suggestion)
```

If `applyAccepted()` returns `success=true`, the suggestion is automatically removed from `assistant_suggestions`.

## Existing Framework Features

Already implemented by `GenericTableAssistant`:

- duplicate prevention
- dismissed suggestion suppression
- suggestion deletion after successful acceptance

No additional implementation is required for these features in issue #783.

## Expected Responsibility of Issue #783

The title-language assistant is expected to implement `applyAccepted()`.

Likely responsibilities:

- load the target `Title` via `target_id`
- validate that the `Title` still exists
- update `Title.language`
- return a success result

The exact acceptance workflow depends on the implementation delivered by issue #782.

## Potential Stale Checks

Issue #783 may need stale suggestion detection depending on the metadata stored by issue #782.

Possible examples:

- title text changed after discovery
- title language changed after discovery
- other language-related source data changed after discovery

The final implementation depends on the discovery metadata generated by issue #782.

## XML Export Impact

The repository currently exports `xml:lang` from `Resource.language` inside `DataCiteXmlExporter::buildTitles()`.

Issue #783 has been confirmed to include DataCite XML export behavior.

This means #783 will likely need to evaluate whether `xml:lang` should be emitted from `Title.language` instead of, or in addition to, `Resource.language`.

The final implementation depends on how title-level language information is stored and applied by issue #782.
